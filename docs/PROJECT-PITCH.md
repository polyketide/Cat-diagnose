# catmed — 猫的循证医疗资料，每个数字都能追到原文

> 项目简介，供开源申请与协作者快速了解。数据以本文件写作时为准，
> 具体计数以 `tools/dr_drill.py leg1` 的实时输出为准。

## 一句话

**它的价值不是"比兽医懂得多"，而是把散落在各篇论文里的证据放在一起、
指出它们互相矛盾的地方、并诚实说出哪里根本没有证据——而且每一个数字都由
机器在每次提交时逐字核回 PubMed。**

## 它是什么

一个**证据整合**项目（不是诊断工具），围绕猫的老年病展开。
当前 180 篇文献、623 条逐字原文摘录、0 条不匹配。包含：

- **分析笔记**（英文，给兽医 / 研究者）——记录哪些结论站得住、哪些证据其实不存在
- **主人指南**（中文，含 PDF）——CKD、全类型淋巴瘤、鼻腔淋巴瘤，不需要医学背景
- **两个受限 agent**——`cat-owner-triage`（面向主人）与 `medical`（面向研究）
- **一个静态站点**——手机可读，无需 git，每页强制免责声明

## 核心纪律：引用契约

**正文是解释，可以争论；文末的原文摘录是证据，逐字未经改写。**
每一条 `> ` 开头的摘录都必须是归档 PubMed 记录的**逐字子串**，
CI 在每次提交时全部核对——**包括陌生人的 PR**。

这不是口号。CI 从**空归档**起步，按知识库引用的 PMID 重新抓取，逐字比对，
再故意破坏一条记录证明检查器还能失败。容灾主张是被持续验证的，不是断言的。

## 为什么它对更多猫主人有用（实用化的五条）

| 维度 | 做法 |
|---|---|
| **受限** | agent 只答知识库覆盖的病、先跑急诊筛查、问到别的直接说"没有验证过的资料"——宁可显得笨，不装什么都懂 |
| **可信** | 主人真正会读的文件，与知识库同受逐字 CI 校验 |
| **可追溯** | 每个数字带行内出处，可追到原句 → 归档记录 → 每次提交核验 |
| **可触达** | 静态站点，手机可读，紧急情况优先提示"先联系兽医" |
| **可批评** | 兽医、生物学家可对文献提批评、走 git 提修改；`CONTRIBUTING.md` 写明贡献规范 |

## 它诚实的地方

这是它和普通宠物科普最不一样的地方——**它明确列出文献里真实的空白**，
并规定：查不到出处的数字宁可标"待核"，也不假装有来源。例如 CKD 指南直说：
早发现能"预测谁会进展"，但"提早干预能否获益"目前**尚未确立**；某个流传的
生存数字因无法定位来源而被标注"出处待核"。

## 起源与隐私

项目源于一只猫的病例。所有临床隐私已脱敏，具名红线检查在每次提交前本地运行
（词表故意不公开，以免泄露它要保护的东西）。MIT 授权。
